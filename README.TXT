Демон запускается командой
pg_sync -c <путь_к_файлу_конфигурации>
на той машине, которая идентифицирована как slave.

Внутри демона работают два потока сихронизации: master->slave и slave->master.
Имена синхронизируемых таблиц указываются в таблице event.syncparam той БД, которая является 
источником синхронизации:
  name - имя таблицы в формате <имя_схемы>.<имя_таблицы>
  key - имя ключевого поля, однозначно идентефицирующего запись в синхронизируемой таблице
  enabled - флаг, определяющий будет ли проводится процедура синхронизации для указанной таблицы

Процесс синхронизации инициируется появлением в таблице event.queue записи, информирующей о
возникновении событие добавления, удаления или модификации записи в исходной таблице:
  event_id - уникальный идентификатор события
  server_id - идентификатор сервера, на котором возникло событие
  event - имя таблицы в формате <имя_схемы>.<имя_таблицы>, на которой возникло событие
  param - значение ключа, идентифицирующего запись, которая была модифицирована
  version - не используется 

Запись о событии добавления, удаления или модификации произдит ф-я event.notify_table_upd.
После обработки события запись из таблицы event.queue удаляется.
